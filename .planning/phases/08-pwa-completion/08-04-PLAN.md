    **Add push_subscriptions table to src/db/schema.ts:**
    Add after the existing `users` table:
    ```typescript
    export const pushSubscriptions = sqliteTable("push_subscriptions", {
      id: integer("id").primaryKey({ autoIncrement: true }),
      endpoint: text("endpoint").notNull().unique(),
      p256dh: text("p256dh").notNull(),
      auth: text("auth").notNull(),
      createdAt: integer("created_at").notNull(), // Unix timestamp ms
    });
    ```

    **Run migration:**
    After adding the table, run `pnpm db:push` to sync the schema to the database.

    **Create POST /api/push/subscribe/route.ts:**
    - Import `db` from `@/db` and `pushSubscriptions` from `@/db/schema`
    - Accept JSON body: `{ endpoint: string, keys: { p256dh: string, auth: string } }`
    - Validate all three fields exist (endpoint, p256dh, auth)
    - Insert into pushSubscriptions with `onConflictDoUpdate` on endpoint (update keys)
    - Return 201 `{ ok: true }`
    - On error, return 400 `{ error: "Invalid subscription" }`

    **Create POST /api/push/send/route.ts:**
    - Import `webpush` from `web-push`, `db` from `@/db`, `pushSubscriptions` from `@/db/schema`
    - Validate `Authorization: Bearer {CRON_SECRET}` (same pattern as /api/scrape)
    - Accept JSON body: `{ title: string, body: string, url?: string }`
    - Set VAPID details: `webpush.setVapidDetails(process.env.VAPID_SUBJECT ?? "mailto:admin@multirotter.com", process.env.NEXT_PUBLIC_VAPID_PUBLIC_KEY!, process.env.VAPID_PRIVATE_KEY!)`
    - Select all subscriptions from pushSubscriptions table
    - For each subscription, call `webpush.sendNotification()` with JSON payload `{ title, body, url: url ?? "/" }`
    - If a subscription returns 410 (expired), delete it from the table
    - Return 200 `{ ok: true, sent: N, failed: N, cleaned: N }`

  <action>
    **Add push event handler to src/app/sw.ts:**
    After `serwist.addEventListeners()`, add:
    ```typescript
    self.addEventListener("push", (event) => {
      if (!event.data) return;
      const data = event.data.json() as { title: string; body: string; url?: string };
      event.waitUntil(
        self.registration.showNotification(data.title, {
          body: data.body,
          icon: "/icons/icon-192x192.png",
          badge: "/icons/icon-192x192.png",
          data: { url: data.url ?? "/" },
        })
      );
    });

    self.addEventListener("notificationclick", (event) => {
      event.notification.close();
      const url = (event.notification.data as { url?: string })?.url ?? "/";
      event.waitUntil(
        self.clients.matchAll({ type: "window" }).then((clients) => {
          for (const client of clients) {
            if (client.url === url && "focus" in client) return client.focus();
          }
          return self.clients.openWindow(url);
        })
      );
    });
    ```

    **Create PushSubscribeButton component:**
    Create `src/components/pwa/PushSubscribeButton.tsx` ("use client"):
    - State: `isSubscribed` (boolean), `isLoading` (boolean)
    - On mount: check if service worker is available AND push subscription exists via `registration.pushManager.getSubscription()`. If subscription exists, set `isSubscribed = true`.
    - `handleSubscribe()`:
      1. Request notification permission: `Notification.requestPermission()`
      2. If denied, alert user and return
      3. Get SW registration: `navigator.serviceWorker.ready`
      4. Subscribe: `registration.pushManager.subscribe({ userVisibleOnly: true, applicationServerKey: urlBase64ToUint8Array(process.env.NEXT_PUBLIC_VAPID_PUBLIC_KEY!) })`
      5. POST subscription to `/api/push/subscribe` with `{ endpoint, keys: { p256dh, auth } }` extracted from `subscription.toJSON()`
      6. Set `isSubscribed = true`
    - `handleUnsubscribe()`:
      1. Get existing subscription, call `.unsubscribe()`
      2. Set `isSubscribed = false`
    - Include the `urlBase64ToUint8Array` helper function (converts VAPID public key from base64 to Uint8Array)
    - Render: button showing "Enable Notifications" or "Notifications Enabled" based on state
    - If `typeof window === "undefined" || !("serviceWorker" in navigator) || !("PushManager" in window)`: return null (SSR safety + browser support check)

    // Trigger push notification if new news items were added
    if (result.newsAdded > 0) {
      try {
        const pushUrl = process.env.NEXT_PUBLIC_SITE_URL
          ? `${process.env.NEXT_PUBLIC_SITE_URL}/api/push/send`
          : "http://localhost:3000/api/push/send";
        await fetch(pushUrl, {

            url: "/news",
          }),
        });
      } catch {

    }

  <name>Task 2: Wire offline fallback into service worker</name>
  <files>src/app/sw.ts</files>
  <action>
    Update `src/app/sw.ts` to handle offline navigation fallback. The Serwist constructor accepts a `fallbacks` option for navigation requests.

    Add to the Serwist constructor options:
    ```
    fallbacks: {
      entries: [
        {
          url: "/offline",
          matcher({ request }) {
            return request.destination === "document";
          },
        },
      ],
    }
    ```

    This tells Serwist: when a navigation request fails (network error), serve the cached /offline page.

    Also ensure "/offline" is included in precache. Add it to the `additionalPrecacheEntries` option of the Serwist constructor:
    ```
    additionalPrecacheEntries: ["/offline"]
    ```

    If Serwist v9 does not support `fallbacks` directly, use the alternative approach: add a `NavigationRoute` with an offline fallback. Check the Serwist v9 API — the key is that when a navigation fetch fails, /offline is served from cache.

    Keep all existing runtimeCaching strategies from Plan 01 intact.
  </action>
  <verify>

  </verify>

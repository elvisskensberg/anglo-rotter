  <files>src/db/schema.ts, src/app/api/threads/[threadId]/rate/route.ts, src/components/thread/ThreadRating.tsx, src/components/thread/OriginalPostBlock.tsx</files>
  <action>
1. Add `threadRatings` table to `src/db/schema.ts`:
   ```
   export const threadRatings = sqliteTable("thread_ratings", {
     id: integer("id").primaryKey({ autoIncrement: true }),
     threadId: integer("thread_id").notNull().references(() => threads.id),
     userId: text("user_id").notNull(), // references user.id (text type from Better Auth)
     score: integer("score").notNull(), // 1-5
     createdAt: integer("created_at").notNull(),
   }, (table) => [
     index("idx_thread_ratings_thread_id").on(table.threadId),
     index("idx_thread_ratings_user_thread").on(table.userId, table.threadId), // unique constraint for one rating per user per thread
   ]);
   ```
   - Also add a unique constraint on (userId, threadId) if Drizzle supports it via `.unique()` — otherwise enforce in application logic

2. Create `src/app/api/threads/[threadId]/rate/route.ts` — POST handler:
   - Get session from auth
   - If no session: return 401
   - Parse body: `{ score: number }` — validate score is integer 1-5
   - Check if user already rated this thread:
     `db.select().from(threadRatings).where(and(eq(threadRatings.threadId, threadIdNum), eq(threadRatings.userId, session.user.id))).limit(1)`
   - If already rated: UPDATE the existing rating with new score
   - If not rated: INSERT new rating
   - After insert/update, recalculate the thread author's star rating:
     a. Get thread author's userId
     b. Query all threadRatings for all threads by that author: `SELECT AVG(score) as avgRating, COUNT(*) as raterCount FROM thread_ratings WHERE thread_id IN (SELECT id FROM threads WHERE author = ?)`
     c. Update user's starRating (round to nearest integer 1-5), ratersCount, and points (sum of all scores)
   - Return 200 with `{ score, averageRating, ratersCount }`

3. Create `src/components/thread/ThreadRating.tsx` ("use client"):
   - Props: `{ threadId: number, currentRating: number, ratersCount: number, points: number }`
   - Import `useSession` from "@/lib/auth-client"
   - If not authenticated: render the existing star display (read-only) using the star SVGs
   - If authenticated: render clickable star icons (1-5) that submit a rating
   - On star click: `fetch(\`/api/threads/\${threadId}/rate\`, { method: "POST", body: JSON.stringify({ score }) })`
   - After submission: update displayed rating locally (optimistic update)
   - Hover effect: highlight stars up to hovered position
   - Use existing star SVGs from /public/icons/star-1.svg through star-5.svg
   - Display: "{ratersCount} raters | {points} points" below the stars

4. Update `src/components/thread/OriginalPostBlock.tsx`:
   - Where the star rating is currently displayed (read-only), integrate ThreadRating component
   - Pass threadId, starRating, ratersCount, and points as props
   - ThreadRating handles both display and interaction
   - This requires adding threadId to OriginalPostBlock props if not already present
   - If OriginalPostBlock is a server component, the ThreadRating (client) can be rendered within it as a client island

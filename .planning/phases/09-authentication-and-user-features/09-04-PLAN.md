  <name>Task 1: Thread creation API + form + forum toolbar link</name>
  <files>src/app/api/threads/route.ts, src/components/forum/NewThreadForm.tsx, src/components/forum/NewThreadForm.module.css, src/app/forum/[forumId]/new/page.tsx, src/components/forum/ForumToolbar.tsx</files>
  <action>
1. Create `src/app/api/threads/route.ts` — POST handler for creating threads:
   - Import `auth` from "@/lib/auth" and get session from request headers: `const session = await auth.api.getSession({ headers: request.headers })` (verify Better Auth's exact API for server-side session retrieval)
   - If no session: return 401 `{ error: "Authentication required" }`
   - Parse request body: `{ title: string, content: string, forumId: string, category?: string }`
   - Validate: title 3-200 chars, content 1-5000 chars, forumId is non-empty string
   - Insert into threads table: `db.insert(threads).values({ id: Date.now(), forumId, title, author: session.user.name || session.user.email, content, excerpt: content.slice(0, 150), category: category || "scoops", viewCount: 0, replyCount: 0, createdAt: Date.now(), lastReplyAt: null, lastReplyAuthor: null })`
   - Also increment the user's postCount: `db.update(user).set({ postCount: sql\`post_count + 1\` }).where(eq(user.id, session.user.id))`
   - Return 201 with `{ id: newThread.id, url: \`/thread/\${newThread.id}\` }`
   - On error: return 500

2. Create `src/components/forum/NewThreadForm.tsx` ("use client"):
   - Import `useSession` from "@/lib/auth-client"
   - Import `useRouter` from "next/navigation"
   - Props: `{ forumId: string }`
   - If not authenticated: render a message "Please log in to post a new thread" with link to /auth/login
   - Form fields: title (text input, required), content (textarea, rows=10, required)
   - On submit: `fetch("/api/threads", { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ title, content, forumId }) })`
   - On success: `router.push(\`/thread/\${data.id}\`)` to navigate to the new thread
   - On error: display error message
   - Style with Rotter aesthetic: table layout, var(--rotter-font-primary), input/textarea with border 1px solid #999

3. Create `src/components/forum/NewThreadForm.module.css`:
   - Same form styling patterns as auth forms

4. Create `src/app/forum/[forumId]/new/page.tsx`:
   - Server component with Rotter page chrome (HeaderBar, BlueNavBar, OrangeNavBar)
   - Renders NewThreadForm with forumId from params
   - Title: "Post New Thread"

5. Update `src/components/forum/ForumToolbar.tsx`:
   - The existing "Post" toolbar icon should link to `/forum/{forumId}/new` instead of being non-functional
   - If ForumToolbar receives forumId as a prop, use it; if not, add forumId prop
   - The "Login" toolbar icon should link to `/auth/login`

1. Create `src/app/api/threads/[threadId]/replies/route.ts` — POST handler:
   - Import `auth` from "@/lib/auth" and get session
   - If no session: return 401
   - Parse params: threadId from URL, body: `{ content: string, parentId?: number }`
   - Validate: content 1-5000 chars, threadId is valid number
   - Verify thread exists: `db.select().from(threads).where(eq(threads.id, threadIdNum)).limit(1)`
   - If thread not found: return 404
   - Get current max postNumber for thread: `db.select({ max: sql\`MAX(post_number)\` }).from(posts).where(eq(posts.threadId, threadIdNum))`
   - Insert into posts: `db.insert(posts).values({ threadId: threadIdNum, parentId: parentId || null, author: session.user.name || session.user.email, content, postNumber: maxPostNumber + 1, createdAt: Date.now() })`
   - Update thread: increment replyCount, set lastReplyAt to now, set lastReplyAuthor
   - Increment user's postCount
   - Return 201 with `{ id: newPost.id, postNumber: newPost.postNumber }`

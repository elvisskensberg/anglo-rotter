  <action>
1. Create `src/components/auth/LoginForm.tsx` ("use client"):
   - Import `signIn` from "@/lib/auth-client"
   - Import `useRouter` from "next/navigation"
   - Import `useState` for error state and loading state
   - Form fields: email (type="email", required), password (type="password", required)
   - On submit: call `signIn.email({ email, password })` (check Better Auth client API — may be `authClient.signIn.email()`)
   - On success: `router.push("/")` (redirect to homepage) or `router.back()` if referrer exists
   - On error: display error message below form (e.g. "Invalid credentials")
   - Loading state: disable submit button, show "Logging in..." text
   - Style with Rotter aesthetic: table-based layout, `var(--rotter-font-primary)` font, `var(--rotter-row-odd)` background, border 1px solid #999
   - Include a link to /auth/register: "Don't have an account? Register"
   - CSS module for form container, input styling, error message

2. Create `src/components/auth/LoginForm.module.css`:
   - `.formContainer` — centered, max-width 400px, padding, background var(--rotter-row-odd)

   - `.submitBtn` — background var(--rotter-row-even), border 1px solid #999, cursor pointer, padding 6px 16px
   - `.error` — color red, font-size var(--rotter-size-sm), margin-top 4px
   - `.link` — color #000099, text-decoration underline

<task type="auto">
  <name>Task 2: Registration page and form component</name>
  <files>src/app/auth/register/page.tsx, src/components/auth/RegisterForm.tsx, src/components/auth/RegisterForm.module.css</files>
  <action>
1. Create `src/components/auth/RegisterForm.tsx` ("use client"):
   - Import `signUp` from "@/lib/auth-client"
   - Import `useRouter` from "next/navigation"
   - Import `useState` for error state and loading state
   - Form fields: username (type="text", required, minLength 3, maxLength 20), email (type="email", required), password (type="password", required, minLength 8)
   - On submit: call `signUp.email({ email, password, name: username })` — Better Auth's name field stores the display name; additionally, if the user table has a custom `username` field, include it in the signup data or update it after account creation
   - IMPORTANT: Better Auth may not support custom fields in signUp directly. If not, the approach is: (a) signUp with email+password+name, (b) after successful signup, make a separate API call to update the username field on the user row. Check Better Auth docs for custom user fields.
   - On success: redirect to /auth/login with a "Registration successful" message (via searchParams), OR auto-login and redirect to "/"
   - On error: display error message (e.g. "Email already registered", "Username taken")
   - Client-side validation: username 3-20 chars alphanumeric, email valid format, password 8+ chars
   - Loading state: disable submit button, show "Creating account..."
   - Same Rotter-style CSS as LoginForm
   - Include a link to /auth/login: "Already have an account? Log in"

2. Create `src/components/auth/RegisterForm.module.css`:
   - Same patterns as LoginForm.module.css

3. Create `src/app/auth/register/page.tsx`:
   - Same layout pattern as login/page.tsx
   - Import RegisterForm, render within Rotter page chrome

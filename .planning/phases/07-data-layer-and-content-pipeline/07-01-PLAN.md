  <action>
    1. Create `src/db/schema.ts` with Drizzle sqliteTable definitions:

       **forums table:**
       - id: text primary key (e.g. "scoops1")
       - name: text not null (e.g. "Scoops & News")
       - slug: text not null unique

       **threads table:**
       - id: integer primary key
       - forumId: text not null, references forums.id
       - title: text not null
       - author: text not null
       - content: text not null (original post body)
       - excerpt: text (first ~200 chars for tooltip)
       - category: text not null default "scoops"
       - viewCount: integer not null default 0
       - replyCount: integer not null default 0
       - createdAt: integer not null (Unix timestamp ms)
       - lastReplyAt: integer (Unix timestamp ms, nullable for threads with no replies)
       - lastReplyAuthor: text

       **posts table:**
       - id: integer primary key autoincrement
       - threadId: integer not null, references threads.id
       - parentId: integer (nullable — NULL = top-level reply)
       - author: text not null
       - content: text not null
       - postNumber: integer not null (sequential reply number)
       - createdAt: integer not null (Unix timestamp ms)

       **news_items table:**
       - id: integer primary key autoincrement
       - headline: text not null
       - source: text not null
       - sourceIcon: text not null (hex color)
       - category: text not null (news/sports/economy/tech)
       - url: text not null default "#"
       - publishedAt: integer not null (Unix timestamp ms)

       **users table:**
       - id: integer primary key autoincrement
       - username: text not null unique
       - memberSince: text not null (date string)
       - postCount: integer not null default 0
       - starRating: integer not null default 1
       - ratersCount: integer not null default 0
       - points: integer not null default 0

       Add indexes: threads.forumId, threads.category, posts.threadId, news_items.category, news_items.publishedAt.

    2. Create `src/db/client.ts`:
       ```typescript
       import { drizzle } from "drizzle-orm/libsql";
       import { createClient } from "@libsql/client";
       import * as schema from "./schema";

       const client = createClient({
         url: process.env.TURSO_DATABASE_URL ?? "file:local.db",
         authToken: process.env.TURSO_AUTH_TOKEN,
       });

       export const db = drizzle(client, { schema });
       ```

    3. Create `src/db/index.ts` barrel export:
       ```typescript
       export { db } from "./client";
       export * from "./schema";
       ```

    4. Create `src/types/forum.ts` with types inferred from Drizzle schema:
       ```typescript
       import type { threads, posts, newsItems, users, forums } from "@/db/schema";

       // Core DB row types (inferred from Drizzle schema)
       export type Thread = typeof threads.$inferSelect;
       export type Post = typeof posts.$inferSelect;
       export type NewsItemRow = typeof newsItems.$inferSelect;
       export type User = typeof users.$inferSelect;
       export type Forum = typeof forums.$inferSelect;

       // API response types (shaped for UI consumption)
       export interface ForumListing {
         id: number;
         title: string;
         author: string;
         date: string;
         time: string;
         lastReplyDate: string;
         lastReplyTime: string;
         lastReplyAuthor: string;
         lastReplyNum: number;
         replyCount: number;
         viewCount: number;
         excerpt: string;
         category: string;
         url: string;
       }

       export interface NewsItem {
         id: number;
         headline: string;
         time: string;
         source: string;
         sourceIcon: string;
         category: string;
         url: string;
       }

       export type NewsCategory = "news" | "sports" | "economy" | "tech";
       ```

       The ForumListing and NewsItem types match the existing seed interfaces so UI components continue working without changes.

    2. Extract `threadId` from params (await Promise).
    3. Query thread: `SELECT * FROM threads WHERE id = ?` — return 404 if not found.
    4. Query posts: `SELECT * FROM posts WHERE threadId = ? ORDER BY postNumber ASC`.
    5. Query author user: `SELECT * FROM users WHERE username = thread.author` (for member info).
    6. Transform into ThreadData-compatible shape:
       - `id`: thread.id
       - `forumId`: thread.forumId
       - `sectionName`: derive from forumId (map: scoops1 -> "Scoops", politics -> "Politics", etc.)
       - `title`: thread.title
       - `post`: { id, author, content, date (formatted), time (formatted), memberSince (from user or "1.1.24"), postCount (from user or 0), starRating (from user or 3), ratersCount, points }
       - `replies`: posts mapped to ReplyTreeItem shape with depth computed from parentId chain (use buildDepthMap BFS approach from thread-seed.ts)
    7. Return `NextResponse.json(threadData)`.
    8. Try-catch with 500 error response.

    1. Import `db` from `@/db`, `newsItems` from `@/db/schema`.

       - `sourceIcon`: row.sourceIcon
       - `category`: row.category
       - `url`: row.url
    5. Return `NextResponse.json({ items: transformedItems })`.
    6. Try-catch with 500 error response.
